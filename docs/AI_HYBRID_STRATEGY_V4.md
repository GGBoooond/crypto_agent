# AI Hybrid Strategy V4 - 止盈止损挂单版

## 1. 核心升级：从轮询平仓到交易所条件单

### V3 的问题
- 止盈止损依赖程序轮询检测，有延迟
- 剥头皮对止盈敏感，轮询频率（10秒）可能错过最佳平仓点
- 网络抖动或程序异常可能导致无法及时止损

### V4 的解决方案
- **开仓时同步挂止盈止损条件单**：交易所级别的触发，毫秒级响应
- **动态调整止盈止损**：每次轮询时让 AI 评估是否需要调整，撤旧挂新

## 3. AI 返回格式变化

### 开仓决策（无持仓时）
```json
{
    "action": "EXECUTE | REJECT",
    "confidence": "HIGH | MEDIUM | LOW",
    "reason": "决策理由",
    "tp_price": 0.12500,  // 具体止盈价格（必填）
    "sl_price": 0.11800   // 具体止损价格（必填）
}
```

### 持仓调整决策（有持仓时）
```json
{
    "action": "ADJUST | HOLD",
    "reason": "调整理由",
    "tp_price": 0.12600,  // 新止盈价格
    "sl_price": 0.11900   // 新止损价格
}
```

## 4. 代码改动清单

### exchange/okx_exchange.py
- 新增 `create_algo_order()` - 创建条件单
- 新增 `cancel_algo_order()` - 取消单个条件单
- 新增 `cancel_all_algo_orders()` - 取消所有条件单

### core/state_store.py
- `Position` 新增字段：
  - `tp_order_id` - 止盈订单ID
  - `sl_order_id` - 止损订单ID
  - `tp_price` - 止盈价格
  - `sl_price` - 止损价格

### strategies/ai_hybrid_strategy.py
- `_build_ai_prompt()` 增加 `is_position_check` 模式
- AI 必须返回具体的 `tp_price` 和 `sl_price`
- 新增 `_handle_position_check()` 处理持仓调整逻辑

### agents/executor_agent.py
- `_open_long()` / `_open_short()` 开仓后自动挂止盈止损单
- 新增 `_place_tp_sl_orders()` - 挂止盈止损条件单
- 新增 `_adjust_tp_sl_orders()` - 调整止盈止损单
- `_close_position()` 平仓时自动取消残余条件单

## 5. OKX 条件单 API

### 创建条件单
```
POST /api/v5/trade/order-algo
{
    "instId": "DOGE-USDT-SWAP",
    "tdMode": "cross",
    "side": "sell",           // 平仓方向
    "posSide": "long",        // 持仓方向
    "ordType": "conditional", // 条件单
    "sz": "10",               // 数量
    "triggerPx": "0.125",     // 触发价格
    "orderPx": "-1",          // -1 = 市价成交
    "triggerPxType": "last",  // 最新价触发
    "reduceOnly": "true"      // 只减仓
}
```

### 取消条件单
```
POST /api/v5/trade/cancel-algos
[{
    "algoId": "xxx",
    "instId": "DOGE-USDT-SWAP"
}]
```

## 6. 优势总结

| 特性 | V3 | V4 |
|------|----|----|
| 止盈止损响应速度 | 依赖轮询（秒级） | 交易所触发（毫秒级） |
| 网络故障保护 | 无（可能无法平仓） | 有（订单在交易所） |
| AI Token 消耗 | 仅开仓时消耗 | 开仓+持仓检查 |
| 止盈止损灵活性 | 固定 ATR 系数 | AI 动态调整 |
| 交易成本 | 仅市价单 | 市价单+条件单 |

## 7. 注意事项

1. **条件单费用**：OKX 条件单本身不收费，触发后按市价单计费
2. **孤儿订单清理**：当止盈/止损其中一个触发后，另一个需要手动取消
3. **价格精度**：AI 返回的价格需要符合交易对精度要求
4. **保证金检查**：条件单在挂单时不锁定保证金，触发时检查
